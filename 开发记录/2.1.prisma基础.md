Prisma 是面向 Node.js/TypeScript 的现代化数据库工具链，核心是类型安全的 ORM，帮你用更简单、更可靠的方式操作数据库。

### 一、一句话定位

它不是传统 ORM（如 Sequelize），而是下一代数据库开发框架，把数据建模、迁移、查询、可视化打包成一套完整工作流。

### 二、三大核心组件

## 1. Prisma Schema（数据模型）

用类自然语言定义表结构，可读性极强：
// schema.prisma
model User {
id Int @id @default(autoincrement())
name String
email String @unique
posts Post[]
}

model Post {
id Int @id @default(autoincrement())
title String
author User @relation(fields: [authorId], references: [id])
authorId Int
}
支持关系（一对多、多对多）、索引、约束
可从现有数据库反向生成（prisma db pull）

## 2. Prisma Client（类型安全查询）

根据 Schema 自动生成 TypeScript 客户端，编译时类型检查，几乎无运行时类型错误：
typescript
运行
// 自动生成的类型，编辑器有完整提示
const user = await prisma.user.findUnique({
where: { email: 'a@example.com' },
include: { posts: true } // 关联查询
})
API 直观、链式调用，不用写复杂 SQL
支持事务、分页、筛选、聚合等

## 3. Prisma Migrate（数据库迁移）

修改 Schema 后，一键生成并执行 SQL 迁移，版本化管理表结构：

# 生成迁移文件并应用到数据库

npx prisma migrate dev --name add_user_bio
迁移文件可手动编辑，支持团队协作同步结构

4. Prisma Studio（可视化管理）
   本地 GUI 工具，直接查看、编辑数据：
   npx prisma studio
   浏览器打开，支持增删改查、筛选、导出

### 三、核心优势

类型安全：TypeScript 强类型，编译期报错，减少线上 BUG
开发高效：自动生成代码，不用手写 DAO/Repository
关系友好：复杂关联查询写法简洁，比原生 SQL 易维护
多数据库支持：PostgreSQL、MySQL、SQLite、MongoDB 等
生态完善：与 NestJS、Next.js、Express 等无缝集成

### 四、与传统 ORM 的区别

表格
特性 Prisma 传统 ORM（Sequelize/TypeORM）
模型定义 独立 Schema 文件，声明式 基于类 / 装饰器，侵入代码
类型安全 自动生成强类型，编译期检查 类型弱，运行时易出错
查询 API 统一、直观、链式 API 复杂，风格不一
迁移 自动生成 + 版本化，可控 配置繁琐，易冲突
学习成本 低，Schema 易读 高，需记忆大量 API
