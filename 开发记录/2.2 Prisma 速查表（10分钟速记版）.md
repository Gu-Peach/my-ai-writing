## 二、Schema 核心语法（模型定义）

### 1. 基础配置（固定开头）

```Plain Text
prisma
// 生成客户端配置
generator client {
  provider = "prisma-client-js"
}

// 数据源配置（以PostgreSQL为例）
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

```

### 2. 字段修饰符（速记）

- **@id**：主键

- **@default(值)**：默认值

- **@unique**：唯一约束，不可重复

- **?**：可选字段，非必填

- **@relation**：关联关系

- **autoincrement()**：自增

- **now()**：默认当前时间

### 3. 标准模型示例

```Plain Text
prisma
model User {
  id       Int     @id @default(autoincrement())
  username String  @unique
  email    String  @unique
  age      Int?    // 可选字段
  createAt DateTime @default(now())
  // 一对多关联
  posts    Post[]
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String?
  // 关联用户
  author    User     @relation(fields: [authorId], references: [id])
  authorId  Int      // 外键字段
  createAt  DateTime @default(now())
}
```

## 三、Prisma Client 初始化（代码开头）

### 1. 基础初始化（简单场景，必记）

```Plain Text
javascript
// 引入客户端
const { PrismaClient } = require('@prisma/client')
// 实例化
const prisma = new PrismaClient()

// 推荐异步函数使用
async function main() {
  // 在这里写CRUD逻辑
}

// 执行并断开连接
main()
  .catch((e) => console.error(e))
  .finally(async () => await prisma.$disconnect())

```

### 2. 全局单例初始化（进阶场景，常用）

用于避免开发环境（如 Next.js、Node.js 热重载）重复创建 Prisma 实例，减少资源消耗，代码如下：

```Plain Text
javascript
import { PrismaClient } from '@prisma/client'

// 全局对象挂载 Prisma 实例，避免重复创建
const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

// 存在则复用，不存在则新建实例（短路求值）
export const prisma = globalForPrisma.prisma ?? new PrismaClient()

// 开发环境挂载到全局，生产环境不挂载
if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

核心作用（速记）：开发环境防重复实例，生产环境正常使用，适配热重载场景。

## 四、常用CRUD操作（高频必背）

### 1. 查询（查）

```Plain Text
javascript
// 单条查询（按唯一字段）
await prisma.user.findUnique({
  where: { id: 1 }
})

// 多条查询
await prisma.user.findMany()

// 带条件、分页、排序
await prisma.user.findMany({
  where: {
    username: { contains: '张' }, // 模糊查询
    age: { gte: 18 } // 大于等于18
  },
  orderBy: { createAt: 'desc' }, // 排序
  skip: 0, // 分页偏移
  take: 10 // 每页条数
})

// 统计数量
await prisma.user.count({
  where: { age: { gte: 18 } }
})

```

### 2. 新增（增）

```Plain Text
javascript
// 单条新增
await prisma.user.create({
  data: {
    username: 'zhangsan',
    email: 'test@123.com'
  }
})

// 批量新增
await prisma.user.createMany({
  data: [
    { username: 'lisi', email: 'lisi@123.com' },
    { username: 'wangwu', email: 'wangwu@123.com' }
  ]
})

```

### 3. 更新（改）

```Plain Text
javascript
// 单条更新
await prisma.user.update({
  where: { id: 1 },
  data: { username: 'zhangsan_new' }
})

// 批量更新
await prisma.user.updateMany({
  where: { age: null },
  data: { age: 20 }
})

```

### 4. 删除（删）

```Plain Text
javascript

// 批量删除

  where: { age: { lt: 18 } }

```

## 七、避坑小提示

1. 修改schema.prisma后，必须执行`npx prisma generate` 同步客户端

2. 建表、改表必须用 `prisma migrate dev`，不要手动改表

3. 主键、唯一字段才能用 `findUnique` 查询

4. Prisma是数据库工具，不是数据库，可搭配PostgreSQL、MySQL、Supabase使用

5. 开发环境（如Next.js）建议用「全局单例初始化」，避免热重载重复创建实例
